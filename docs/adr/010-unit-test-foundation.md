# ADR-010: 引入单元测试体系（unittest 标准库）

- **日期**：2026-06-24
- **状态**：已采纳，被 ADR-012 修订

## 背景

项目运行到 v1.0 后，发现：

1. ADR-008 / ADR-009 都是真实存在但未自动检测的 bug
2. 改 _parse_precondition 时只能手动跑 experiments/ 验证，开发反馈周期长
3. 后续 v1.1（并行）/ v2.0（Swarm）/ v3.0（记忆）演进如果没有测试基线，每次改动都是"盲改"

## 决策

在项目根目录建 tests/ 目录，使用**标准库 unittest**（零外部依赖，与项目零重型依赖原则一致）：

- 命名：test_<模块名>.py
- 运行命令：python -m unittest discover -s tests -v
- 与项目其他规范一致：测试不依赖 mock 框架外的任何外部包

## 关联 ADR

- ADR-002（核心 Agent 内核自主实现）：自研内核必须配自研测试
- ADR-005（核心分层重构）：core/ 与 multi_agent/ 都需要测试覆盖
- ADR-006（关系驱动引擎）：v1.0 演进路线必须有测试基线

## 后果

- 后续 PR 修改任何核心函数时，开发者可 0.01s 跑完测试确认未破坏
- ADR-009 类的潜在 bug 可在 CI 阶段捕获

## 修订说明

本 ADR 的"测试基线"目标被 ADR-012 重新定义为"风险点守护"，覆盖率指标不再追求。详见 ADR-012。
