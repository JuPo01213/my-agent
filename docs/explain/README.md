# docs/explain/ — 代码走读与解释文档

## 用途
存放模型与用户对话过程中产生的**走读式解释文档**，与 `docs/report/` 的调研报告区分：

| 目录 | 内容 |
|------|------|
| `docs/report/` | 外部调研、最佳实践、架构方案（**面向决策**） |
| `docs/explain/` | 内部代码走读、运行日志复盘、bug 复盘（**面向理解**） |

## 文件命名规范
- 代码走读：`代码走读-<文件名>-<日期>.md`
- 真实跑记录：`<日期>-<实验名>.log`（**注意：日志文件在 `experiments/`，不走读文档**
- 道歉信/复盘：`道歉信-<日期>.md` / `复盘-<主题>-<日期>.md`

## 必填章节（走读文档）
- 入口：用户问题的原文或一句话总结
- 当前状态：mock / 半真 / 真跑（必须明确）
- 走读路径：用户跟随阅读的步骤顺序
- 每步必须含：目的 / 行号定位 / 代码片段 / 解释
- 关键设计点：作者的设计意图
- 常见疑问：FAQ 形式

## 写文档的铁律
- 写完后**必须用 Read 验证**文件真的落盘
- 引用代码必须用 `file:///` 链接 + 行号锚定
- 不得"声称写完"但文件不存在
