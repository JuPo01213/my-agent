# 开发指南

本文档描述 my-agent 项目的开发流程、代码规范和最佳实践。

## 开发环境

- Python 3.10+
- 标准库 unittest（零外部依赖）
- Git 版本控制

## 代码规范

- 所有文本文件写入必须显式声明 UTF-8 编码
- 代码变更前须先提交，提交时必须附上中文说明
- 编写代码必须包含详细注释
- 对话内容均须使用中文，任何时候都不要使用 emoji

## 开发流程

1. 修改代码前先提交当前状态
2. 实现功能或修复问题
3. 更新相关文档（`docs/structure.md`、README 等）
4. 运行测试：`python -m unittest discover -s tests -v`
5. 运行同步检查点：`python .github/skills/project-initializer/scripts/sync_checkpoint.py`
6. 提交变更，附上中文说明

## 重构原则

- 当架构需要根本性改变时，完全重构优于兼容改造
- 测试阶段不必背负历史包袱，宁可一次改干净也不要新旧两套并存
- 重构后要主动更新所有调用点，同步更新文档和 ADR

## 测试原则

- 风险点优先，不是覆盖率优先
- 每个测试都该回答：守的是项目逻辑还是 Python 标准行为？
- 未来哪个改动会因这个失败被打回？
- 是否 mock 出来假行为？

## 实验规范

- 所有真实实验必须把 stdout/stderr 全部重定向到日志文件
- 路径约定：`experiments/<日期>-<实验名>.log`
- 禁止伪造执行结果与运行数据
