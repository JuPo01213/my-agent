## 一、执行摘要

本报告综合 7 个搜索技能的调研结果，系统回答两个核心问题：

1. **做自己的智能体需要收集哪些信息？**
2. **成熟的最佳实践是什么？**

**核心结论**：构建一个可用的智能体，需要的“信息”远不止一个 API key 和一段 prompt。它需要：清晰的**能力定义**、可扩展的**工具生态**、分层的**记忆机制**、工程化的**规划推理能力**、纵深防御的**安全对齐体系**、多维度的**评估指标**、按需选择的**多智能体协作模式**，以及生产就绪的**部署运维基础设施**。

## 三、成熟架构模式

### 3.1 架构演进路径

```
Level 1: 简单调用链
  User → LLM + Tool Schema → 单次 function call → 结果返回

Level 2: ReAct / 推理-行动循环
  LLM 交替执行推理（Thought）与工具调用（Action）
  支持多步、可观察、可纠错

Level 3: 规划增强
  - CoT（思维链）：显式分步推理
  - ToT（思维树）：并行探索多路径
  - ReWOO：将规划与执行分离，降低 token 消耗
  - Plan-and-Solve：先生成计划，再逐步执行

Level 4: 多智能体协作
  - 层级架构：Orchestrator → Workers
  - 对等架构：Peer agents 协商分配
  - 流水线架构： specialised agents 串行处理
```

### 3.3 四种架构模式速查表

| 模式 | 适用场景 | 复杂度 | 代表工具 |
|------|----------|--------|----------|
| **单工具链（Pipeline）** | 线性任务，如文档摘要、翻译 | ★☆☆ | 裸 OpenAI API + JSON Schema |
| **ReAct Agent** | 需要多步推理 + 工具调用 | ★★☆ | LangGraph, CrewAI |
| **多智能体协作（Swarm）** | 复杂工作流，如代码审查 + 测试 | ★★★ | AutoGen, CrewAI Teams |
| **RAG + Agent 混合** | 企业知识库问答 | ★★☆ | LlamaIndex + LangChain |

**选择建议**：从单工具链开始，当且仅当出现“需要动态选择工具”或“需要多步自我修正”时，升级到 ReAct。

## 四、代码实现与工具选型

### 4.1 最小可用智能体代码示例

#### 4.1.1 最简 ReAct 智能体（单文件，无外部框架）

```python
"""
最小可用 ReAct Agent —— 单文件、无重型框架依赖。
运行方式：export OPENAI_API_KEY=...; python minimal_react.py
"""
import json
import os
import re
from openai import OpenAI

client = OpenAI()

# 工具定义
TOOLS = [
    {
        "type": "function",
        "function": {
            "name": "calculator",
            "description": "计算数学表达式，例如 '12 * (4 + 6)'",
            "parameters": {
                "type": "object",
                "properties": {
                    "expression": {"type": "string", "description": "数学表达式字符串"}
                },
                "required": ["expression"],
            },
        },
    },
    {
        "type": "function",
        "function": {
            "name": "search",
            "description": "搜索内部知识库并返回最相关的片段",
            "parameters": {
                "type": "object",
                "properties": {
                    "query": {"type": "string", "description": "检索查询词"}
                },
                "required": ["query"],
            },
        },
    },
]

def call_tool(name: str, arguments: dict):
    if name == "calculator":
        try:
            return str(eval(arguments["expression"]))  # noqa: S307
        except Exception as e:
            return f"计算错误: {e}"
    elif name == "search":
        return f"[检索结果] 关于 '{arguments['query']}' 的模拟片段"
    return "未知工具"

SYSTEM_PROMPT = """你是一个智能助手。请使用以下工具解决问题。
每轮回复必须严格按以下格式：
Thought: 你的思考
Action: 工具名称，参数为 JSON
Observation: 工具返回结果
Final Answer: 最终答案"""

def run_react(user_query: str, max_rounds: int = 5):
    messages = [
        {"role": "system", "content": SYSTEM_PROMPT},
        {"role": "user", "content": user_query},
    ]
    for _ in range(max_rounds):
        resp = client.chat.completions.create(
            model="gpt-4o",
            messages=messages,
            tools=TOOLS,
            tool_choice="auto",
        )
        msg = resp.choices[0].message
        messages.append(msg)
        if msg.tool_calls:
            for tc in msg.tool_calls:
                result = call_tool(tc.function.name, json.loads(tc.function.arguments))
                messages.append({
                    "role": "tool",
                    "tool_call_id": tc.id,
                    "content": result,
                })
            continue
        if "Final Answer:" in (msg.content or ""):
            return msg.content
    return "未能在最大轮次内完成。"

if __name__ == "__main__":
    print(run_react("公司今年营收增长率是多少？请先搜索再计算。"))
```

#### 4.1.2 状态化智能体（LangGraph 简化思路）

```python
from dataclasses import dataclass, field
from typing import List, Optional
import json
from openai import OpenAI

client = OpenAI()

@dataclass
class AgentState:
    task: str
    thought: str = ""
    action: str = ""
    observation: str = ""
    history: List[dict] = field(default_factory=list)
    final_answer: Optional[str] = None

class StatefulAgent:
    def __init__(self, model: str = "gpt-4o"):
        self.model = model

    def step(self, state: AgentState) -> AgentState:
        prompt = f"""当前任务: {state.task}
历史记录: {json.dumps(state.history, ensure_ascii=False)}
请输出:
Thought: ...
Action: search 或 calculator 或 finish
Observation: (如已执行工具则填写结果)"""
        resp = client.chat.completions.create(
            model=self.model,
            messages=[{"role": "user", "content": prompt}],
            temperature=0,
        )
        text = resp.choices[0].message.content or ""
        thought_match = re.search(r"Thought: (.*)", text)
        action_match = re.search(r"Action: (\w+)", text)
        if thought_match:
            state.thought = thought_match.group(1).strip()
        if action_match:
            state.action = action_match.group(1).strip()
        if state.action == "finish":
            state.final_answer = text
        return state

    def run(self, task: str, max_steps: int = 5):
        state = AgentState(task=task)
        for _ in range(max_steps):
            state = self.step(state)
            state.history.append({
                "thought": state.thought,
                "action": state.action,
            })
            if state.final_answer:
                break
        return state

if __name__ == "__main__":
    agent = StatefulAgent(model="gpt-4o-mini")
    result = agent.run("计算 3 的 5 次方加 20")
    print(result.final_answer)
```

#### 4.2.2 递归字符分割实现

```python
from typing import List

def recursive_text_splitter(
    text: str,
    chunk_size: int = 500,
    chunk_overlap: int = 50,
    separators: List[str] = None,
) -> List[str]:
    """递归字符分割：优先按段落、句子、单词拆分"""
    if separators is None:
        separators = ["\n\n", "\n", "。", "！", "？", " ", ""]
    chunks = []
    def split(text, depth):
        if len(text) <= chunk_size:
            chunks.append(text)
            return
        separator = separators[depth] if depth < len(separators) else ""
        if separator:
            parts = text.split(separator)
            current_chunk = ""
            for part in parts:
                if len(current_chunk) + len(part) <= chunk_size:
                    current_chunk += part + separator
                else:
                    if current_chunk:
                        chunks.append(current_chunk)
                    current_chunk = part + separator
            if current_chunk:
                chunks.append(current_chunk)
        else:
            chunks.append(text[:chunk_size])
    split(text, 0)
    return chunks
```

## 八、最佳实践总结

### 8.1 技术最佳实践

1. **保持简单**：从单 LLM call + 检索开始，证明不够再升级。
2. **工具优先**：投入与 HCI 同等精力设计 Agent-Computer Interface（ACI）。
3. **透明可观测**：每一步 tool call、状态转换、延迟都要可追踪。
4. **上下文工程**：Summarization 压缩历史、Memory 跨会话记忆、Prompt caching 降本。
5. **容错与安全**：重试、退避、PII 过滤、人工审批、最大迭代次数——这些不是可选项。
6. **框架选型原则**：
   - 快速原型/简单流程 → LangChain `create_agent`
   - 复杂有向图/多智能体 → LangGraph
   - 低代码企业部署 → CrewAI
   - 统一工具接入 → MCP
7. **不要框架成瘾**：先理解底层，再决定是否用框架。

### 8.2 工程最佳实践

1. **分层防御**：安全不是单点，而是“提示层 + 运行时层 + 基础设施层”的纵深防御。
2. **渐进式自主**：从人工审批为主，逐步过渡到条件自动执行，再到全自主运行。
3. **评估先行**：在第一个 tool 接入时，同步写入 10 条 golden case。
4. **成本意识**：智能体通常需要多步推理，token 消耗是单轮问答的 3-10 倍。
5. **数据治理**：知识库上线前必须清洗、去重、打标签。

### 8.3 组织最佳实践

1. **团队匹配**：1 人追求单文件闭环，5 人追求可编排，10 人+ 追求私有化合规。
2. **Prompt 即代码**：版本控制、Code Review、CI/CD 都要覆盖 prompt 和工具 schema。
3. **用户教育**：首次对话展示能力边界，内置示例，降低用户预期错位。
4. **持续运营**：智能体上线只是开始，持续评估、监控、迭代才是核心竞争力。
