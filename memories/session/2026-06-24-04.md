## 关键决策与结论
- **SSE 事件到气泡类型映射**：thought→thinking、action→tool、observation→附加到最近 tool 气泡、reply/final→agent、error/stopped/end→对应类型
- **observation 合并策略**：将 observation 附加到最近一个未消费 observation 的 tool 气泡，复用 action 卡片展示完整 IO
- **气泡位置缓存**：最多保留 20 个位置，超过则丢弃最早的，避免内存膨胀
- **打字机仅作用于 agent 气泡**：thought/tool/system 气泡直接展示，提升信息密度
- **气泡类型与样式解耦**：通过 `eventToBubbleType` 映射函数集中处理，便于扩展

## 踩坑记录
- **v-html 注入风险**：用户输入和 LLM 输出都需先 HTML 转义再做 Markdown 替换，使用 `escape` 函数处理
- **气泡高度估算**：DOM 渲染前无法获取真实高度，placeBubble 使用 100px 估算值，必要时可用 nextTick 后重定位
- **SSE 停止与打字机竞态**：stopCurrent 需同时清理 eventSource 与 typingTimer，避免停止后光标仍闪烁
