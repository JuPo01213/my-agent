## 踩坑记录
- **问题 1**：PowerShell `>` 重定向到文件时，Python 多次 print 只保留最后一行
  **根因**：本机 PowerShell I/O 重定向在阻塞管道下行为异常（已用 `echo A B C > file; cat file` 复现仅剩 C）
  **解决方案**：改用 Python `logging.FileHandler` 内部写文件，stderr 同步输出，彻底绕开 shell 重定向
- **问题 2**：第一次跑用 `python -u > log 2>&1` 后只看到 "SOME FAILED"
  **根因**：同问题 1；SOME FAILED 是脚本最后一行 stdout，被 shell 截断成单行
  **解决方案**：与问题 1 一并解决
- **问题 3**：脚本启动慢、ReAct 跑 3 步每步 13s+，单次运行 40s+
  **根因**：StepFun step-3.7-flash 单次响应慢；默认 max_iter=8 意味着单 Agent 最多 8 次 LLM 调用
  **解决方案**：脚本中显式 `a.max_iter = 3` 覆盖 YAML 默认 8（用完后立即还原等价），单 Agent 控制在 5-21s
