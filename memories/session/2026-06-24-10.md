## 关键决策与结论

### 决策 1：A2 测试用 unittest 标准库（不引入 pytest）
- **理由**：与项目零重型依赖原则一致（ADR-002）
- **代价**：API 略冗长（必须继承 `TestCase` + `self.assertEqual`），但换来零装包

### 决策 2：测试 mock 不用 unittest.mock.patch 装饰器
- **理由**：用 `_make_client(responses=[...])` 顺序返回 mock response，更接近真实 LLM 行为
- **替代**：可改用 `unittest.mock.MagicMock` 但会让测试代码变成"框架地狱"
- **当前选择**：直接写 mock 类，更显式易读

### 决策 3：ADR-009 修复用"前移分支"而非"加判断"
- **理由**：解析器按"特殊性 → 一般性"匹配是通用原则；加判断会重复扫描字符串
- **副作用**：把 `import re` 从函数内部移到模块顶部（之前是 lazy import，现在无必要）

### 决策 4：ADR-010 测试覆盖目标按优先级
- 已覆盖：`_parse_precondition`（P0） + `run_loop`（P0）
- 未覆盖：Blackboard / ControlShell / Command / RelationshipEngine.run（后续 P0/P1）
- **理由**：建立测试基线是 P0，但不必一次全做，按风险等级分批

## 踩坑记录

### 踩坑 1：sync_checkpoint.py 输出 UnicodeEncodeError
- **现象**：PowerShell 默认 GBK 编码无法编码 emoji（脚本用 `\U0001f504` 同步图标）
- **解决**：设 `$env:PYTHONIOENCODING='utf-8'`
- **教训**：项目规则"任何时候都不要使用 emoji"与 sync_checkpoint.py 矛盾——脚本层面违反规则，应在脚本源里替换为中文（不在本轮范围内）

### 踩坑 2：测试自身 bug：_MockResponse 传参错误
- **现象**：12 个测试全部 ERROR（`TypeError: _MockResponse.__init__() got an unexpected keyword argument 'content'`）
- **根因**：第一版误把 `content=` 当成 `_MockResponse` 的构造参数，实际是 `_MockMessage` 的
- **修复**：用 5 次 `multi_replace_string_in_file` 把所有 `_MockResponse(content=` 改成 `_MockResponse(_MockMessage(content=`
- **教训**：写 mock 类要严格按"OpenAI 协议三层结构：Response → Choice → Message"建模

### 踩坑 3：PowerShell heredoc（`@""..."@`）失效
- **现象**：用 PowerShell 的 `Add-Content -Value @"..."@` 追加 ADR-009/010 到 md 文件，命令无报错但内容未写入
- **根因**：PowerShell here-string 在复杂嵌套 + 中文环境下不可靠
- **解决**：改用 `replace_string_in_file` 在 ADR-008 末尾追加新 ADR（更可控）
- **教训**：涉及多行 Markdown 内容追加时，优先用工具而非 shell heredoc

### 踩坑 4：replace_string_in_file 误删大量内容
- **现象**：第一次更新 `docs/structure.md` 时，oldString 把"当前归档"列表全部选中但 newString 没包含 → 列表被删
- **修复**：第二次 replace 把列表恢复 + 加 `tests/` 段
- **教训**：replace 旧字符串时必须先用 read_file 看上下文，避免"以为还在但实际没了"
