## 关键决策与结论
- 事件格式采用**中性 trace**（kind/step/data），core 层不包含前端协议信息
- 前端语义事件全部由 frontend.adapter 在 bus.emit 后翻译，保证 core 与前端零耦合
- EventBus 需兼容两种字段（type 和 kind），因为 adapter 输出用 type，中性 trace 用 kind
- 实验 KPI 证明全链路可跑通：35 条总线事件、3 Agent 激活、最终 facts 含 3 个 result

## 踩坑记录
- **问题**：KPI 断言失败，EventBus.count() 只检查 type 字段
  **根因**：bus.emit 直接透传中性 trace 时用 kind，而 adapter 聚合后生成 type，count() 只匹配 type
  **解决方案**：count() 同时检查 e.get("type") 和 e.get("kind")
- **问题**：首次实验 KPI 中 llm.final 匹配失败
  **根因**：实验里把中性 trace 的 kind="final" 写成了 type="llm.final"
  **解决方案**：统一用 kind 字段匹配中性 trace，或让 adapter 统一输出 type 前缀事件
